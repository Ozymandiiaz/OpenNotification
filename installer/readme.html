Reliable Response Notification

Installation Guide


Welcome to Reliable Response Notification. This installer will guide you through the process of installing and configuring Reliable Response Notification. Before you begin, please read this document closely and prepare all prerequisits.


Database Installation

Reliable Response Notification requires a SQL relational database. Currently, only Oracle, version 8 or higher, and PostgreSQL, version 7 or higher, are supported. You will need to have a database and database instance for use. Notification requires privileges to add, modify and delete records, but not to create databases or tables. Please have available the hostname of the database server, the instance name of the database, and a username and password we can use to login.


Email Support

Reliable Response Notification requires an outgoing SMTP server for sending emails to recipients. Simply supply the hostname of the SMTP server during the installation process.


Reliable Response Notification can also accept notifications through email. It can either retrieve emails sent to POP server, find the recipient in the subject or a command to pass or confirm an existing notification. To use this option, you will need a POP email account. Notification can also serve as its own SMTP server. This involves more setup, and generally requires the support of the Reliable Response professional services organization.

Instant Messenger Support

Reliable Response Notification supports sending notifications through two of the most popular Instant Messenging systems, Jabber and AOL. To use these systems, you must create an account which Notification may use. Both IM systems allow you to create accounts with any standard IM client.
